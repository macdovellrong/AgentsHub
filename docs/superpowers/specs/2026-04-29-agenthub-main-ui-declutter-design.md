# AgentHub 主界面降噪设计

## 目标

把主界面从调试 dashboard 收敛为多 Agent 聊天协作界面。日志和转发记录仍然保留，但不再常驻占用主界面空间。

## 日志入口

运行记录、raw log 和过滤器从右侧常驻面板移到按需打开的日志面板。顶部提供“日志”按钮。点击后打开右侧抽屉/覆盖面板，里面保留现有 runs 列表、profile/status 过滤、raw.log 预览。关闭后回到聊天主界面。

日志仍然服务于排错、复盘和审计，不参与日常聊天主流程。

## 转发入口

移除常驻“转发控制”面板。转发从聊天消息上触发：点击某条消息的“转发”后，输入框上方显示引用条，用户输入 `@codex ...` 或选择目标 Agent 后发送。发送时创建并立即发送 forward，底层继续写入 `forwards.jsonl` 和 `events.jsonl`。

如果用户没有引用消息，普通 `@agent` 发送仍直接写入对应 Agent PTY，不创建 forward 记录。

## 主界面布局

- 左侧：工作区、Agent 列表、角色配置。
- 中间：聊天气泡、引用条、输入框。
- 右侧：当前工作区的 Agent 终端。
- 顶部：打开工作区、日志、刷新、在线数量。

任务看板本版暂时保留常驻，后续可以用同样方式移到按钮/抽屉。

## 非目标

- 不删除底层 runs/forwards/tasks 存储。
- 不实现 Agent 输出自动入聊天。
- 不实现多条消息合并引用或富文本引用。
