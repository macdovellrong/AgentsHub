# AgentHub 聊天输入与气泡界面设计

## 目标

本版优化中央聊天区的人工交互体验，只做三件事：`@` 智能选择 Agent、去掉 `@` 发送时的多余包装文本、把中央事件流改成聊天软件式气泡。Agent 输出自动进入聊天区暂不实现。

## `@` 智能选择

用户在中央输入框输入 `@` 后显示候选 Agent 列表。继续输入会按 profile id、名称、别名过滤，例如 `@c` 匹配 Codex。候选项显示名称、id 和在线状态。用户可以用鼠标点击，或用方向键选择后按 Enter。Esc 关闭候选框。

选中候选项后，输入框中插入规范 token：`@codex `。发送时仍沿用现有路由逻辑，离线 Agent 不自动启动，只提示用户先启动。

## 发送内容

中央聊天输入发送到 PTY 时，只发送用户正文和一个回车。`@codex hello` 最终写入 Codex 终端的是 `hello\r`。不再每次拼接 profile 的 role prompt，避免终端里出现 `AgentHub profile instructions`、`User message` 等多余文本。

## 聊天气泡

中央区域继续使用 `.agenthub/events.jsonl` 作为数据源，但渲染改成聊天气泡：

- `user_message` 靠右显示。
- `agent_output` 和 `agent_forward` 靠左显示。
- `error` 使用居中的系统提示。
- 会话、任务、编排事件使用较轻的系统气泡。

每条气泡保留时间、发送者/类型和“转发”操作。输入区固定在底部，事件更新后滚动到底部。

## 非目标

- 不自动把 Agent 原始终端输出写入聊天区。
- 不实现消息编辑、撤回、已读状态。
- 不实现富文本、Markdown 渲染或文件附件。
